# MaoqiuPlayer Project Prompt

项目英文名：`MaoqiuPlayer`  
中文名称：`毛球播放器`  
GitHub 仓库名：`maoqiu-player`  
默认媒体包后缀：`.mqp`  
自定义文件头 Magic：`MAOQIU_PLAYER_ENC_V1`

## 产品定位

MaoqiuPlayer 是一款跨平台本地媒体播放器，主要面向视频和图片的本地播放、媒体库管理和私有媒体包播放。

软件界面应参考主流播放器交互方式，整体看起来像普通媒体播放器，而不是明显的安全工具或文件处理工具。首页不突出媒体包生成、访问口令、底层存储细节等高级能力。

## 首页原则

首页显示：

- 毛球播放器 / MaoqiuPlayer
- 最近播放
- 媒体库概览
- 本地视频入口
- 本地图片入口
- 播放列表入口
- 打开文件入口
- 设置入口

首页不显示：

- 媒体包生成的大按钮
- 底层存储或访问控制细节
- 高级工具入口的显眼宣传

## UI 文案原则

优先使用播放器语境：

- 媒体包
- 私有媒体包
- 生成媒体包
- 导入媒体包
- 打开媒体
- 媒体库
- 缓存管理
- 文件校验
- 高级工具

避免在用户界面中过度突出底层技术词。内部代码可以保留 `packer`、`encryptor`、`decryptor`、`crypto`、`key_info` 等技术命名，但 UI 层不直接暴露这些词。

## 媒体包入口

媒体包生成功能入口固定为：

```text
设置 -> 高级设置 -> 高级工具 -> 媒体包管理 -> 生成媒体包
```

用户进入该页面后，可以选择视频或图片，生成 `.mqp` 或自定义后缀媒体包。页面文案使用“媒体包”“私有媒体包”“导出包”等表述。

## 播放行为

当用户打开普通视频或图片时，直接进入对应播放或查看页面。

当用户打开 `.mqp` 或带有 `MAOQIU_PLAYER_ENC_V1` 文件头的媒体包时，应用在后台完成识别、临时展开和播放，不在主界面上强调底层处理过程。
